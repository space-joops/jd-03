# 백로그: 자랑 기능 바이럴화 방안

자랑(공유) 기능을 텍스트 복사에서 "공유하고 싶은 이미지 + 유입 루프"로 키우기 위한 브레인스토밍과 로드맵. (2026-07-18 브레인스토밍)

## 문제 인식

기존 자랑 기능은 텍스트 → 클립보드였다. 받는 사람 입장에서 밋밋하고(이미지 없음), 보는 사람이 게임으로 들어올 경로도 없었다. 바이럴은 ① 공유하고 싶은 결과물 ② 공유하고 싶어지는 타이밍 ③ 본 사람이 할 수 있는 행동, 세 가지가 맞물려야 한다.

## 방안

### 1. 이미지 카드 — "텍스트"를 "짤"로 ✅ 구현됨 (1단계)

스프라이트·캔버스 렌더러가 이미 있으므로 오프스크린 캔버스에 카드를 그려 PNG로 뽑는다. 서버 불필요.

- 1080×1080 픽셀 아트 카드: 계열 색 프레임(트레이딩 카드 감성) + 궤도 링 + 대형 도트 펫 + 스탯
- 공유 3단 폴백: Web Share(파일, 모바일 공유 시트) → 클립보드(이미지) → 다운로드
- **실감 비유 카피**: 누적 kg → "경차 1.5대 분량" (볼트/자전거/냉장고/통신위성/경차 구간)
- 구현: `src/lib/game/bragImage.ts`

남은 아이디어 (미구현):

- 세로형 1080×1920 스토리용 프리셋
- 카드 컨셉 변형: 다마고치 기기 프레임, 관제 콘솔 보고서 스타일
- 펫의 한마디(계열·기분 기반 랜덤 대사)를 카드에 — 카드마다 달라 재공유 유발

### 2. 감정 고점에서 공유 제안 ✅ 구현됨 (2단계)

"자랑 버튼을 눌러야 공유"가 아니라 자랑스러운 순간에 게임이 먼저 제안:

- 트리거(우선순위순): 수동 조종 신기록(🏆) > 진화(✨) > 대형 잔해 견인 성공(🪝) > 임무 7/30/100일차 도달
- 배너 형태, 12초 후 자동 소멸, 닫기 가능
- 신기록은 전용 스코어 카드(`renderSortieCard`) — "30초에 {kg}kg" + 도전 유도 카피 + #수동조종챌린지
- 신기록 추적: `GameState.sortieBestKg` (구세이브 백필 포함), 경신 시에만 🏆 로그

남은 아이디어: 진화 순간 카드에 이전 형태 → 새 형태 비교 연출

### 3. 도전장 URL — 유입 루프 (3단계, 미구현)

바이럴의 핵심은 본 사람이 할 게 있어야 한다는 것:

- 미니게임 점수를 URL 쿼리에 인코딩(`?challenge=187`) — 정적 앱이어도 서버 없이 도전장 성립
- 도전장 링크로 온 사람은 **펫 없이 미니게임 1판 데모 출격** 가능하게 → 종료 후 "네 알을 받아 제대로 키워봐" 전환
- 카드 구석에 QR 코드(클라이언트 생성) — 스크린샷 경유 유입까지 커버

### 4. 동적 링크 미리보기 (4단계, 서버 필요, 미구현)

- Vercel 배포 전제로 `next/og` 엣지 함수(`/api/og?stage=...&kg=...`) → 카톡에 링크만 보내도 개인화 카드가 미리보기로 뜸. 카톡 공유 체감이 가장 큰 항목
- 주간 리더보드, 초대 보상은 백엔드가 생긴 뒤 재검토

## 카피 훅 메모

- 사회공헌 감성이 공유 부담을 낮춘다: "지구 저궤도가 ○○만큼 깨끗해졌습니다" + #케슬러신드롬청소반
- 숫자보다 비유가 리트윗을 탄다 (비유 변환기 채택됨)
- 계열 분기(스피드/당김/균형)는 사실상 수집 요소 — "내 카드는 이벤트 호라이즌" 자랑이 성립하도록 카드에 계열을 시각화 (프레임 색 채택됨)

## 로드맵 현황

| 단계 | 내용 | 상태 |
| --- | --- | --- |
| 1 | 캔버스 PNG 카드 + Web Share (계열 프레임, 비유 카피) | ✅ 완료 |
| 2 | 감정 고점 공유 프롬프트 + 미니게임 스코어 카드 | ✅ 완료 |
| 3 | 도전장 URL + 데모 출격 + QR | 대기 |
| 4 | next/og 동적 링크 미리보기 | 대기 (Vercel 전제) |

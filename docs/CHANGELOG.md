# 개발 이력 (CHANGELOG)

STELLAPET의 기능 개발 연혁을 마일스톤 단위로 정리한다. 상세 커밋은 `git log`, 규칙·수치의 현재 상태는 [REQUIREMENTS.md](REQUIREMENTS.md) 참조. 버전(v0.2.x)은 `package.json`이 단일 원본이며 배포 시점에 올린다.

## 1. MVP — 코어 루프 (`04aea35`)

궤도 청소 다마고치의 뼈대 구현.

- 단계 흐름: 알 품기 → 지상 육성(먹이·훈련·보살핌) → 라이드셰어 발사 등록(5분 윈도우) → 13초 발사 시퀀스 → 궤도 임무
- 순수 함수 엔진(`engine.ts`): `tick`/`act`/`catchUp`(부재중 정산, 상한 8시간)
- 픽셀 아트 캔버스(`PixelView`): 지상/발사/궤도 씬, 코드로 그린 스프라이트
- localStorage 저장, 텍스트 자랑 카드, 4단계 진화(수거량 임계값)
- 이후 로그 증가 시 액션 버튼이 밀리는 레이아웃 수정 (`db9ede9`)

## 2. 궤도 이벤트와 진화 계열 분기 (`fb1f763`)

궤도 임무에 변주 추가.

- ☄ 유성우(45초, 조우 ×2.5·충돌 ×2) / 🌞 태양 플레어(60초, 조우 ×0.5·기분 저하 ×2) / 📍 대형 잔해 견인 제안(30초 제한, 당김 비례 성공률)
- 2단계(1,000kg)부터 스탯 성향에 따라 **스피드/당김/균형** 계열 분기 — 단계 이름과 진화 보너스가 갈림
- 구버전 세이브 백필 체계 도입 (`storage.ts`)

## 3. 문서 체계 (`3c51240`, `339ccbd`)

- `docs/REQUIREMENTS.md`: 전체 규칙·밸런스 수치 명세. "수치 변경 시 문서 동기화" 원칙 수립
- README 전면 재작성, `docs/ARCHITECTURE.md`(코드 구조)·`docs/CONTRIBUTING.md`(초심자용 기여 가이드) 신설

## 4. PWA (`46350bf`, `7abf848`, `4b1faee`, `8ceec15`)

- 웹 앱 매니페스트(전체화면 실행, 세로 고정) + EGG 스프라이트 기반 아이콘(192/512/마스커블/apple-touch) — PNG를 스크립트로 직접 인코딩
- 수제 서비스 워커: 네비게이션 네트워크 우선(온라인이면 항상 최신), 해시 자산 캐시 우선, 오프라인 셸 폴백
- 푸터 앱 설치 버튼(상시 노출, 미지원 환경엔 플랫폼별 안내 토스트) + `v{버전}` 표기
- SW 캐시 버전을 `package.json version`과 연동(`/sw.js?v=`) — 버전 업 배포만으로 캐시 세대교체

## 5. 수동 조종 미니게임 (`3076dc4` → `114fc39` → `cfe0d92`)

jd-02(스페이스 죽스)의 게임 로직을 참조한 액티브 조작 모드.

- 조종 액션(추진제 5, 쿨다운 60초)으로 30초 미니게임 진입, `settleSortie`로 본편 정산(수거량·기분·진화 연동), 조종 중 본편 틱 정지
- 클로저 게임 루프(update/draw 분리, dt 상한), 후한 획득/짠 피격 판정, 자석, 꿀꺽 연출, 무적·화면 흔들림
- 전체 화면 전환: fixed 오버레이 + Fullscreen API, 논리 해상도를 화면비로 동적 결정 (`114fc39`)
- 조작 개편 (`cfe0d92`): **3단 분사 조그셔틀**(드래그 거리 = 분사 단계, 연료 게이지), 우주 관성(마찰 감쇠 + 최소 표류 속도), 벽 반동(×−0.8), 잔해 사방 진입

## 6. 사운드 (`c5ca4a9`)

jd-02의 Web Audio 신시사이저 참조 — 오디오 파일 0개.

- 이벤트 로그 kind → 효과음 자동 매핑(gain 코인/warn 버즈/evo 팡파르/sys 비프/info 블립)으로 전 분야 커버
- 발사 럼블, 미니게임 먹기·피격·출격/복귀 징글, 분사 3단 엔진음 루프
- 자동재생 정책 대응(첫 제스처 언락), 푸터 🔊/🔇 뮤트(localStorage 저장)

## 7. 바이럴 공유 시스템 (1856c5f → 45f4f98 → 0f8c941 → 063beef)

자랑 기능을 "텍스트 복사"에서 "이미지 + 유입 루프"로. 브레인스토밍·로드맵은 [backlog/viral-brag.md](backlog/viral-brag.md).

- **1단계** — 1080×1080 픽셀 아트 자랑 카드(계열 색 프레임, 실감 비유 카피 "경차 1.5대 분량") + 공유 3단 폴백(Web Share → 클립보드 → 다운로드)
- **2단계** — 감정 고점 공유 프롬프트(신기록 > 진화 > 견인 성공 > 기념일), `sortieBestKg` 신기록 추적, 전용 스코어 카드
- **3단계** — 도전장 URL(`/c?kg&n`) + 카드 QR, 도전장 방문자는 가입 없이 데모 출격 → 승패 비교 → 알 배정 전환
- **4단계** — `/c` 랜딩의 개인화 og 메타 + `/api/og`(next/og, Galmuri 폰트·스프라이트 div 그리드)로 카톡·X 링크 미리보기에 개인화 카드 노출 (Vercel 서버리스)

## 8. 리더보드 — Supabase 연동

공유 Supabase 프로젝트(`jd03_` 접두사 스키마) 기반 첫 서버 기능.

- 보드 3종: 주간 신기록(월요일 KST 리셋) / 누적 수거량 / 명예의 전당(주별 1위 뷰)
- 익명 인증(기기별 uid) + RLS(읽기 공개·쓰기 본인만), 최초 1회 공개 동의 후 자동 동기화
- `/rank` 페이지: 탭 3종, 캐릭터 클릭 → 업적 모달, 내 순위 하이라이트·랭크 카드 자랑(공유 시점 박제)
- 엔진에 주간 신기록 추적(`weekKey`, `sortieWeek`/`sortieWeekBestKg`) 추가
- env 미설정 시 기능 전체가 조용히 꺼짐 — 오프라인 게임성 유지

## 1. 준비물 설치하기

세 가지가 필요합니다. 이미 있다면 다음 단계로 가세요.

### Node.js (자바스크립트 실행기)

이 프로젝트를 돌리는 엔진입니다. [nodejs.org](https://nodejs.org/ko)에서 **LTS 버전**(왼쪽의 추천 버전)을 받아 설치하세요. 설치가 됐는지 터미널에서 확인합니다:

```bash
node --version   # v20 이상이면 OK
npm --version    # 숫자가 나오면 OK (Node와 함께 설치됩니다)
```

> **터미널이 뭔가요?** 명령어를 글자로 입력하는 창입니다. 맥은 "터미널" 앱, 윈도우는 "PowerShell"을 검색해 열면 됩니다. 이 문서의 `code 블록` 안 명령은 전부 여기에 붙여넣고 Enter를 누르면 됩니다.

### Git (코드 버전 관리 도구)

수정 내역을 기록하고 다른 사람과 코드를 주고받는 도구입니다. [git-scm.com](https://git-scm.com/downloads)에서 설치하세요. 확인:

```bash
git --version
```

### 코드 에디터

[VS Code](https://code.visualstudio.com/)를 추천합니다. 무료이고, 이 프로젝트가 쓰는 TypeScript를 잘 이해합니다.

## 2. 프로젝트 받아오기

### 처음 한 번만

GitHub에서 이 저장소 페이지 오른쪽 위의 **Fork** 버튼을 누르세요. 내 계정 아래에 복사본이 생깁니다 — 원본을 망칠 걱정 없이 마음껏 수정할 수 있는 내 작업장입니다.

그다음 내 컴퓨터로 내려받습니다 (`<내 아이디>`를 본인 GitHub 아이디로 바꾸세요):

```bash
git clone https://github.com/<내 아이디>/jd-03.git
cd jd-03
npm install        # 프로젝트가 쓰는 라이브러리들을 설치 (1~2분 걸립니다)
```

`npm install`은 처음 한 번(그리고 누군가 라이브러리를 추가했을 때)만 하면 됩니다.

## 3. 게임 실행해 보기

```bash
npm run dev
```

터미널에 `http://localhost:3000` 이 보이면 성공입니다. 브라우저에서 그 주소를 열어 보세요 — 게임이 뜹니다! 🥚

이 상태에서 **코드를 수정하고 저장하면 브라우저가 자동으로 새로고침**됩니다. 끄고 싶으면 터미널에서 `Ctrl+C`.

> 팁: 모바일 화면 느낌으로 보려면 브라우저에서 `F12` → 왼쪽 위 휴대폰 모양 아이콘(기기 툴바)을 누르세요.

## 4. 코드는 어디에 있나요?

파일이 많아 보이지만, 처음에는 **딱 3개만** 기억하면 됩니다:

| 파일 | 담당 | 이런 걸 바꾸고 싶다면 |
| --- | --- | --- |
| `src/lib/game/engine.ts` | **게임 규칙·밸런스** | 쿨다운, 진화 조건, 이벤트 확률, 수거량 공식, 로그 문구 |
| `src/components/Game.tsx` | **화면(UI)** | 버튼, 상태 바, 배너, 푸터, 문구·색상 |
| `src/components/SortieGame.tsx` | **미니게임** | 조작감, 잔해 스폰, 분사·관성 물리 (`TUNE` 상수) |

나머지가 궁금해지면 [ARCHITECTURE.md](ARCHITECTURE.md)(코드 구조)와 [REQUIREMENTS.md](REQUIREMENTS.md)(게임 규칙 명세)를 읽어 보세요.

> 📚 **더 깊이 배우고 싶다면**: [`study/`](../study/README.md) 폴더에 13장짜리 인수인계 가이드가 있습니다. 웹 개발 기초(HTML/CSS/JS/React)부터 이 프로젝트의 게임 엔진·캔버스·사운드·PWA·리더보드까지 코드와 함께 친절하게 설명합니다.

## 5. 첫 수정 실습: 밸런스 숫자 바꿔보기

가장 안전하고 재미있는 첫 기여는 숫자 하나 바꿔보기입니다. 같이 해 봅시다.

**목표: 먹이 쿨다운을 6초에서 3초로 줄여 보기**

1. VS Code로 프로젝트 폴더를 엽니다 (`파일 → 폴더 열기`)
2. `src/lib/game/engine.ts` 를 열고 `COOLDOWNS` 를 찾습니다 (`Ctrl+F` 검색)

```ts
export const COOLDOWNS: Record<string, number> = {
  incubate: 3_000,
  feed: 6_000,     // ← 이 줄!
  ...
```

3. `6_000`(밀리초 = 6초)을 `3_000`으로 바꾸고 저장합니다
4. 브라우저로 돌아가면 — 먹이 버튼 쿨다운이 3초가 되어 있습니다 🎉

> `6_000`의 밑줄(`_`)은 그냥 읽기 좋으라고 넣는 자릿수 구분입니다. `6000`과 같습니다.

미니게임 손맛을 바꾸고 싶다면 `SortieGame.tsx` 상단의 `TUNE` 상수를 만져 보세요 — 주석으로 각 숫자의 역할이 적혀 있습니다.

**⚠️ 한 가지 규칙**: 밸런스 수치를 바꿨다면 [REQUIREMENTS.md](REQUIREMENTS.md)에서 같은 수치를 찾아 함께 고쳐 주세요. 코드와 문서의 숫자가 어긋나면 다음 사람이 헷갈립니다.

## 6. 수정한 게 문제없는지 확인하기

PR을 보내기 전에 두 가지를 확인합니다:

```bash
npm run build
```

이 명령이 **코드 문법 검사(린트)와 타입 검사**를 함께 수행합니다. 빨간 에러 없이 끝나면 통과입니다. 에러가 나면 겁먹지 말고 메시지를 읽어 보세요 — `src/components/Game.tsx:123` 처럼 **파일 이름과 줄 번호**를 알려 줍니다. 대부분 오타이거나 타입이 안 맞는 경우입니다.

그리고 **직접 플레이**해 보세요. 바꾼 부분이 실제로 의도대로 동작하는지, 다른 곳이 깨지지 않았는지 눈으로 확인하는 것이 가장 좋은 테스트입니다.

> 게임을 처음부터 다시 시작하고 싶으면 화면 맨 아래 "초기화"를 누르거나, 시크릿 창에서 열면 됩니다 (저장이 localStorage에 있어서요).

## 7. 커밋하고 PR 보내기

### 브랜치 만들기

`main`(기본 줄기)을 직접 고치지 말고, 작업용 가지를 만듭니다:

```bash
git checkout -b feed-cooldown-tune   # 작업 내용을 짐작할 수 있는 이름이면 됩니다
```

### 커밋 (수정 내역 기록)

```bash
git add .                            # 바꾼 파일 전부를 기록 대상으로
git commit -m "먹이 쿨다운을 6초에서 3초로 단축"
```

커밋 메시지는 이 프로젝트처럼 **한국어로, 무엇을 왜 바꿨는지**가 드러나게 써 주세요. `git log --oneline`으로 기존 메시지들을 보면 감이 옵니다.

### PR(Pull Request) 열기

```bash
git push origin feed-cooldown-tune
```

푸시하면 터미널이나 GitHub의 내 fork 페이지에 **"Compare & pull request"** 버튼이 나타납니다. 누르고 다음을 적어 주세요:

- **무엇을** 바꿨는지 (한두 문장이면 충분)
- **왜** 바꿨는지 (예: "먹이 텀이 길어서 초반이 심심했어요")
- 화면이 바뀌었다면 스크린샷 한 장

완벽하지 않아도 괜찮습니다. PR은 "이거 어때요?"라고 대화를 시작하는 방법이지, 완성품 제출이 아닙니다. 리뷰에서 수정 요청이 오는 건 자연스러운 과정이니 부담 갖지 마세요.

## 8. 자주 막히는 곳 (FAQ)

**`npm install`에서 에러가 나요**
Node 버전이 낮을 수 있습니다. `node --version`이 v20 미만이면 [nodejs.org](https://nodejs.org/ko)에서 LTS로 다시 설치하세요. 그래도 안 되면 에러 메시지 마지막 20줄과 함께 이슈로 질문해 주세요.

**`localhost:3000`이 이미 사용 중이래요**
다른 프로그램(아마 예전에 켜둔 dev 서버)이 그 포트를 쓰고 있습니다. 그 터미널을 찾아 `Ctrl+C`로 끄거나, `npm run dev -- -p 3001`로 다른 포트를 쓰세요.

**소리가 안 나요**
브라우저 정책상 첫 클릭/터치 후부터 소리가 납니다. 화면을 한 번 누른 뒤 확인해 보세요. 푸터의 🔇가 켜져 있지 않은지도 확인!

**예전 화면이 계속 보여요 (수정이 반영 안 되는 느낌)**
`npm run start`(프로덕션)로 봤다면 서비스 워커 캐시일 수 있습니다. 개발 중에는 `npm run dev`를 쓰고, 꼬였다면 브라우저 개발자도구(F12) → Application → Service Workers → Unregister 후 새로고침하세요.

**게임 상태를 처음으로 되돌리고 싶어요**
푸터의 "초기화" 버튼, 또는 F12 → Application → Local Storage에서 `stellapet-save-v1` 삭제.

**git이 꼬였어요 / 뭘 잘못 눌렀는지 모르겠어요**
`git status`를 먼저 실행해 보세요 — 지금 상태를 알려 줍니다. 커밋 전 수정을 전부 되돌리려면 `git checkout .` (주의: 저장 안 된 수정이 사라집니다). 해결이 안 되면 상태 출력을 복사해서 질문해 주세요. git은 원래 처음에 다 어렵습니다.
